JVM Basics

1️ What is JDK, JRE, JVM?

* JVM (Java Virtual Machine)

- JVM is a virtual machine that runs Java programs
- It converts Java bytecode into machine code (that your computer understands)
- It also handles:
  - Memory management
  - Garbage collection

JVM is the engine that runs Java programs

---

* JRE (Java Runtime Environment)

- JRE provides everything needed to run Java programs
- It includes:
  - JVM
  - Libraries (like predefined Java classes)

JRE = JVM + Libraries to run programs

---

* JDK (Java Development Kit)

- JDK is used to develop Java programs
- It includes:
  - JRE
  - Compiler ("javac")
  - Development tools

JDK = JRE + tools to create programs

---
Relationship Summary

JDK = JRE + Development Tools
JRE = JVM + Libraries

---

What is Bytecode?

- When you write Java code (".java" file), it is compiled using "javac"
- The output is a ".class" file

This ".class" file contains bytecode

* Key Points

- Bytecode is not machine code
- It is an intermediate code
- It is understood only by the JVM

* Bytecode is the code that JVM understands and executes

---

3 ️What Does “Write Once, Run Anywhere” Mean?

Java follows the principle of Write Once, Run Anywhere (WORA).

This means that once you write and compile a Java program, the generated bytecode can run on any system (Windows, Mac, Linux) without changing the code. The only requirement is that the system must have a compatible JVM installed.

This is possible because the JVM acts as a bridge between the bytecode and the operating system. Instead of converting Java code directly into machine-specific instructions, Java first converts it into bytecode, which can be executed by any JVM regardless of the platform.

Write the code once, and run it on any device with JVM — no need to rewrite for each OS.
